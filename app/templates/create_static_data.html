About Status & Priority

Statuses define the workflow state of bugs. Role Access Type determines which user group can use a status. Priorities help categorize bug severity levels.

Quick Guide

Statuses

Define the lifecycle stages of bugs (e.g., Open, In Progress, Resolved, Closed).

Role Types

DEVELOPER: For dev team, TESTER: For QA team, ALL: Both can use.

Priorities

Categorize bug urgency (Critical, High, Medium, Low). Helps in prioritization.
